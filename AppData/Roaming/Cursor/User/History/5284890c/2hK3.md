# Deployment Guide - Baby Steps

## Step 1: Update Firebase Config (Use Environment Variables)

1. Open `src/firebase/config.js`
2. Replace with your actual Firebase config values from Firebase Console
3. **OR** use environment variables (recommended for production)

## Step 2: Push to GitHub

```bash
# Initialize git (if not already done)
git init

# Add all files
git add .

# Commit
git commit -m "Initial commit - Ready for deployment"

# Create repo on GitHub, then:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git branch -M main
git push -u origin main
```

## Step 3: Deploy to Netlify

1. Go to [netlify.com](https://netlify.com) → Sign up/Login
2. Click **"Add new site"** → **"Import an existing project"**
3. Connect to GitHub → Select your repo
4. Build settings (auto-detected):
   - **Build command:** `npm run build`
   - **Publish directory:** `dist`
5. Click **"Deploy site"**
6. Wait for build to complete
7. Copy your Netlify URL (e.g., `https://your-app.netlify.app`)

## Step 4: Update Firebase Auth Domain

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Select your project
3. **Authentication** → **Settings** → **Authorized domains**
4. Click **"Add domain"**
5. Add your Netlify URL (e.g., `your-app.netlify.app`)
6. Click **"Add"**

## Step 5: Deploy Firebase Security Rules

1. In Firebase Console → **Firestore Database** → **Rules**
2. Copy content from `firestore.rules` file
3. Paste into Firebase Rules editor
4. Click **"Publish"**

## Step 6: Update Firebase Config in Code (If using env vars)

1. In Netlify → Your site → **Site settings** → **Environment variables**
2. Add variables:
   - `VITE_FIREBASE_API_KEY`
   - `VITE_FIREBASE_AUTH_DOMAIN`
   - `VITE_FIREBASE_PROJECT_ID`
   - `VITE_FIREBASE_STORAGE_BUCKET`
   - `VITE_FIREBASE_MESSAGING_SENDER_ID`
   - `VITE_FIREBASE_APP_ID`
3. Redeploy site

## Step 7: Test Your App

1. Visit your Netlify URL
2. Test authentication
3. Test Firestore operations
4. Check browser console for errors

## Troubleshooting

- **CORS errors:** Check Firebase Auth authorized domains
- **Firestore errors:** Verify security rules are published
- **Build fails:** Check Netlify build logs
- **404 on refresh:** Netlify redirects should handle this (already in `netlify.toml`)
