# Quick Reference - Roster Management App

## Directory Structure

```
roster-app/
├── src/
│   ├── index.html              # Main HTML entry
│   ├── index.js                # Bun server (port 3000)
│   ├── styles/
│   │   ├── main.css            # Military olive theme
│   │   ├── components.css      # Cards, buttons, forms
│   │   └── responsive.css      # Mobile/desktop
│   ├── js/
│   │   ├── app.js              # Core controller
│   │   ├── ui.js               # UI rendering
│   │   ├── data.js             # Data management
│   │   ├── roster.js           # Roster operations
│   │   ├── assignments.js      # Assignment logic
│   │   ├── storage.js          # localStorage/files
│   │   └── backup.js           # Autosave/backup
│   └── data/
│       └── roster-data.json    # Data template
├── dist/                       # Build output
├── bin/                        # Executable output
├── bun.toml                    # Bun config
└── package.json                # Package config
```

## Commands

### Development
```bash
bun run dev        # Start dev server (localhost:3000)
bun start          # Same as above
```

### Build
```bash
bun run build      # Build to dist/
bun run compile    # Create bin/app.exe
```

## File Purposes

| File | Purpose |
|------|---------|
| `src/index.html` | Main HTML entry point |
| `src/index.js` | Bun HTTP server |
| `src/js/app.js` | Application controller |
| `src/js/ui.js` | UI rendering |
| `src/js/data.js` | JSON data management |
| `src/js/roster.js` | Personnel roster logic |
| `src/js/assignments.js` | Assignment algorithms |
| `src/js/storage.js` | localStorage/file I/O |
| `src/js/backup.js` | Autosave & backups |
| `src/styles/main.css` | Global styles (olive theme) |
| `src/styles/components.css` | Component styles |
| `src/styles/responsive.css` | Responsive design |

## Module Dependencies

```
app.js → ui.js, data.js, roster.js, storage.js, backup.js
backup.js → storage.js (dynamic)
```

## Data Flow

1. **Load**: localStorage → data.js → app.js → ui.js
2. **Save**: ui.js → roster.js/assignments.js → data.js → localStorage
3. **Backup**: backup.js → storage.js (every 5 min)
